## Analysis

### What Worked
1. **SFT converged fast** — 2 epochs is the sweet spot, val loss improved steadily
2. **DPO perfectly separates chosen/rejected** — 100% reward accuracy
3. **Total training time ~63 min** on H200 — very manageable
4. **Unsloth 4-bit** worked flawlessly — only ~6-7 GB VRAM used
5. **TLJH environment fixes** (torch cleanup, TF disable) are now battle-tested

### What Didn't Work
1. **GRPO didn't improve itemset accuracy** — F1 reward near zero throughout
2. **Model doesn't use `<think>` during GRPO** — thinking_reward always 0.0
3. **Quick inference test failed** — JSON parse error, missed itemsets
4. **DPO might be too easy** — 100% accuracy = rejected examples are trivially bad

### Root Causes
- **GRPO F1 near zero:** The model generates valid JSON structure but wrong itemset content. After SFT+DPO it learned the format perfectly but can't generalize to new datasets without CoT
- **No `<think>` in GRPO:** The GRPO prompt format probably doesn't include the system prompt that instructs CoT, OR the model learned not to think during the generation phase
- **DPO too easy:** Real LLM failures are often blatantly wrong (empty output, malformed JSON) — the model doesn't learn subtle quality improvements

## Next Steps

### For v2 Training
1. **Evaluate v1 first** — run `eval_finetuned_model.py` to get real F1/P/R numbers
2. **Consider skipping GRPO** — SFT+DPO alone may suffice; GRPO added 45 min for minimal gain
3. **If keeping GRPO:** 
   - Include system prompt in GRPO dataset prompts
   - Increase max_completion_length to 3072+ (room for `<think>` + JSON)
   - Increase to 500+ steps
   - Try higher LR (1e-5 instead of 5e-6)
4. **Harder DPO negatives:** Mix in near-miss failures (correct format, wrong counts/items)
5. **More SFT data:** Generate from more pipeline runs (currently 314 examples)
