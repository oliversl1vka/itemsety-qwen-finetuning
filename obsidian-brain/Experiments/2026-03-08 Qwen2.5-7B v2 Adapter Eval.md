## Analysis

### What worked:
1. ✅ Adapter loading via `FastLanguageModel.from_pretrained()` — 392 LoRA params confirmed
2. ✅ Council was RIGHT — `merged_4bit_forced` destroyed quality. Adapter mode recovers `<think>` + JSON
3. ✅ On small datasets (10×3), F1=78.6% — proves SFT training succeeded for simple cases
4. ✅ Repetition penalty fixes the looping (27/30 finish vs 2/15) — but creates new problems

### What didn't work:
1. ❌ `no_repeat_ngram_size=3` — NEVER use for structured/template output. Destroys JSON generation.
2. ❌ `repetition_penalty=1.3` too aggressive — penalizes the repeated tokens that JSON requires
3. ❌ 314 SFT examples insufficient for reliable termination behavior
4. ❌ SFT-only (no DPO/GRPO) — model lacks preference signal for "complete good output" vs "loop"

### Root Cause:
The model is **undertrained** — it learned FORMAT (✅ `<think>`, ✅ JSON schema) but not TERMINATION. On simple datasets it sometimes finishes naturally; on anything complex it loops. Inference parameters cannot fix this:
- No penalty → loops forever
- With penalty → garbled output
- The solution is **better training**, not better inference config
