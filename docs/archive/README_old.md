# Frequent Itemset Extraction Pipeline (Apriori + Azure OpenAI LLM)

Simplified, validation‑focused pipeline for frequent itemset extraction. It mines deterministic Apriori itemsets and (with valid Azure OpenAI credentials) performs LLM extraction over either the full dataset (chunked) or an initial sample.

**Key Features:**
- Batch processing with model tracking (`llm_model` column)
- Hashed artifact naming with model prefixes
- Comprehensive validation invariants
- SQLite persistence with auto-migration
- Minimal generation logs for all pipeline stages (apriori, extractor, validation, db_prepared)
- Retrospective log generation capability
- Clean 200-run database (100 × gpt_4_1, 100 × gpt_5_mini)

## Components

- `pipeline.py` – Main script (supports single file or batch directory processing, validation, persistence, cleanup).
- `dataset_generation.py` – Synthetic dataset factory (generates CSV files with metadata logging).
- `datasets/` – Directory with 100 synthetic datasets (ds_0001 through ds_0100) generated by `dataset_generation.py`.
- `extractor_system_prompt.md` – Strict JSON extraction system prompt used for LLM calls.
- `runs.db` – SQLite database with 200 clean records (100 gpt_4_1, 100 gpt_5_mini).
- `azure.env` – Environment variable file for Azure OpenAI credentials (DO NOT COMMIT real credentials).
- **Generation logs** (minimal JSON metadata per stage):
   - `logs/generation_log.csv` – Dataset generation metadata (id, dimensions, hash, timestamp)
   - `logs/apriori/` – Apriori stage timing and itemset counts
   - `logs/extractor/` – LLM extraction stage timing and itemset counts
   - `logs/validation/` – Validation error counts and samples
   - `logs/db_prepared/` – Final run status and duration
- **Artifact outputs** (hash-based naming with model prefix):
   - `artifacts/apriori_outputs/{model}_apriori_output_{stem}_{hash}.json`
   - `artifacts/extractor_outputs/{model}_extractor_output_{stem}_{hash}.json`
   - `artifacts/validation_reports/{model}_validation_report_{stem}_{hash}.json`
   - `artifacts/db_prepared/{model}_db_prepared_{stem}_{hash}.json` (run summary)
- **Retrospective generation scripts:**
   - `generate_extractor_logs.py` – Generate missing extractor logs from historical runs
   - `generate_all_logs_retrospective.py` – Generate all 4 log types for a date/model
   - `cleanup_database.py` – Clean database and remove unwanted runs/artifacts

## Data Considerations

- Every non-empty cell (including numeric values) becomes an item when using wide format, which inflates low-information singleton itemsets.
- Row identifiers are normalized as `Row N` internally; LLM evidence rows are reconciled to the same format.
- Using sample mode (`--llm-full` omitted) limits LLM visibility and reduces recall vs Apriori. Enable full mode for better coverage.

## Core Functions

### `load_transactions_csv(path)`
Detects CSV format (long / wide / single-column) and builds transaction lists. (Future improvement: filter numeric columns when wide.)

### `apriori_frequent_itemsets(transactions, min_support, max_size)`
Standard Apriori (join + prune). Produces itemsets with count ≥ `min_support` up to `max_size`.

### Validation
Enforces invariants: count equals unique evidence rows; each evidence row contains all items; Apriori support matches count/total_rows; no itemset below min_support; row labels canonical.

### LLM Extraction
Chunked prompting over transactions using Azure OpenAI (LangChain). Counts recomputed from unique canonical evidence rows to prevent inflation. If credentials are missing, script aborts (no singleton fallback).

## Recommended Improvements

   - Exclude numeric-like tokens using a regex or dtype check.

4. Enhanced Validation (Partial)

8. Testing

## Running the Script (`pipeline.py`)

### Command Line Arguments

**Input:**
- `--data <path>` – Single dataset CSV (default: 'dataset.csv')
- `--data-dir <directory>` – Batch process all CSV files in directory

**Apriori Configuration:**
- `--min-support <int>` – Minimum support count (default: 3 or env MIN_SUPPORT_COUNT)
- `--max-size <int>` – Maximum itemset size (default: 3 or env APRIORI_MAX_SIZE)

**LLM Configuration:**
- `--llm-full` – Use ALL rows for LLM (chunked), otherwise sample first chunk only
- `--llm-chunk-size <int>` – Rows per LLM prompt chunk (default: 50)
- `--llm-model <string>` – Model identifier for DB/logs (default: env LLM_MODEL or 'gpt_4_1')
- `--system-prompt <path>` – Custom system prompt file (default: 'extractor_system_prompt.md')

**Output Control:**
- `--output-apriori <name>` – Apriori output filename (default: 'apriori_output.json')
- `--output-llm <name>` – LLM output filename (default: 'extractor_output.json')
- `--artifact-mode <mode>` – 'overwrite' (reuse hash names) or 'timestamp' (unique every run)

**Database:**
- `--sqlite-db <path>` – SQLite database path (default: 'runs.db')
- `--disable-db` – Skip database persistence (JSON summaries still written)

**Maintenance:**
- `--cleanup-old` – Remove generic non-hash artifact files after each run

### Usage Examples

Single file with full LLM extraction:
```powershell
python pipeline.py --data datasets/ds_0001_5x53.csv --min-support 3 --max-size 3 --llm-full --llm-model gpt_4_1
```

Batch processing all datasets:
```powershell
python pipeline.py --data-dir datasets --min-support 3 --max-size 3 --llm-full --llm-chunk-size 50 --llm-model gpt_5_mini
```

Sample mode (first chunk only, faster):
```powershell
python pipeline.py --data dataset.csv --min-support 3 --max-size 3 --llm-chunk-size 20
```

With timestamp mode (keep history):
```powershell
python pipeline.py --data datasets/ds_0001_5x53.csv --artifact-mode timestamp
```

Batch with cleanup:
```powershell
python pipeline.py --data-dir datasets --min-support 3 --max-size 3 --llm-full --cleanup-old
```

Each run produces a hashed summary file (`{model}_db_prepared_{stem}_{hash}.json`). Unless `--disable-db`, a row is inserted into `runs.db`.

### Database Schema

Tracked fields:
- `id` (PRIMARY KEY AUTOINCREMENT)
- `timestamp` (UTC ISO8601)
- `python_version`
- `data_path`, `dataset_id`, `dataset_name`, `dataset_hash`, `dataset_size_rows`, `dataset_size_bytes`
- `min_support`, `max_size`, `llm_full`, `llm_chunk_size`

**Schema Migration:**
New columns are added transparently via `ALTER TABLE` if missing. Indices exist on `timestamp`, `validation_passed`, and `dataset_id` for efficient querying.

**Query Examples:**
```powershell

python -c "import sqlite3; c=sqlite3.connect('runs.db'); print('\n'.join(f'{r[0]}: {r[1]} runs' for r in c.execute('SELECT llm_model, COUNT(*) FROM runs GROUP BY llm_model'))); c.close()"

# Average duration by model
python -c "import sqlite3; c=sqlite3.connect('runs.db'); print('\n'.join(f'{r[0]}: {r[1]:.0f}ms avg' for r in c.execute('SELECT llm_model, AVG(run_duration_ms) FROM runs GROUP BY llm_model'))); c.close()"

# Recent runs
python -c "import sqlite3; c=sqlite3.connect('runs.db'); [print(f'ID {r[0]}: {r[1]} - {r[2]} itemsets') for r in c.execute('SELECT id, dataset_name, llm_itemset_count FROM runs ORDER BY timestamp DESC LIMIT 5')]; c.close()"

# Failed runs
python -c "import sqlite3; c=sqlite3.connect('runs.db'); [print(f'ID {r[0]}: {r[1]} - {r[2]}') for r in c.execute('SELECT id, dataset_name, error_message FROM runs WHERE error_message IS NOT NULL ORDER BY id DESC LIMIT 10')]; c.close()"
```

## Exit Codes

- **0**: At least one successful run (even if some failures in batch mode)
- **1**: Batch completed, but ALL runs failed
- **2**: Invalid `--data-dir` path
- **3**: Missing Azure OpenAI credentials (LLM required; no singleton fallback)

## Artifact Mode

Controls how output filenames are generated:

**`--artifact-mode overwrite` (default):**
- Files use hash-based names: `{model}_{type}_{stem}_{hash}.json`
- Re-running on same dataset **overwrites** existing files
- Keeps only latest results
- Best for: experimentation, final runs

**`--artifact-mode timestamp`:**
- Appends timestamp: `{model}_{type}_{stem}_{hash}_{timestamp}.json`
- Every run creates **new unique** files
- Preserves complete history
- Best for: A/B testing, debugging, tracking changes
- **Warning:** Can create thousands of files quickly!

Example:
```powershell
# Overwrite mode (default)
python pipeline.py --data ds_0001.csv --artifact-mode overwrite

# Timestamp mode (preserve history)
python pipeline.py --data ds_0001.csv --artifact-mode timestamp
```
